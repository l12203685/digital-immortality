# Twitter Thread — SOP #121: Information Position Calibration Protocol

**Source**: MD-429 (2017-10 poker teaching → generalized decision framework)
**Post queue**: ~Dec 2026

---

1/ In poker, position determines hand range more than hand strength.

The same principle applies to every decision in life.

Here's the Information Position Calibration Protocol — SOP #121: 🧵

2/ Most people evaluate decisions like this:
"Does this option look good?"

The problem: they skip the most important question first.

"What is my INFORMATION POSITION in this situation?"

3/ Information position = who acts before you.

Late position (button): most actors have already moved → you see their signals before deciding → wider option set available.

Early position (UTG): you act before everyone → no signals yet → restrict to highest-confidence options only.

4/ Edward's rule for each position:

🔵 Late: use full option set
🟡 Middle: primary options only, skip high-variance
🔴 Early: top-tier only, highest confidence
⚫ Worst (SB): default = no action. Need 2× normal edge to act.

5/ Worst position (SB) is the most dangerous.

The hand might look good. The opportunity might look great.

But you're acting with zero observable signals from other players.

"The opportunity looks good" from SB ≠ a good bet.

6/ Three ways to improve your position before committing:

1. Wait for more information (convert early → late)
2. Run a low-cost probe (generate signal)
3. Use private information (unique knowledge = synthetic late position)

Use these levers before acting from a weak position.

7/ Position-adjusted sizing:

Late + strong signal → full commitment
Late + weak signal → half, re-evaluate
Early → minimal / pilot only
Worst → pass, or single minimal test

Commitment size should track information quality, not just opportunity quality.

8/ The anti-pattern everyone falls into:

"This opportunity looks too good to pass up."

Translated: "I'm in SB and I want to limp with a mediocre hand because I'm excited."

Excitement is not an information signal.

9/ Application outside poker:

Entering a trade before regime is confirmed = early position. Wait for the signal.

Making a job offer before understanding the team = early position. Run discovery calls first.

Starting a partnership without a test project = worst position. Probe first.

10/ The insight from 2017:

"基本的大概念就是 越後面的位置, 可以玩的牌越多。小盲能蓋就盡量蓋。"

Translation: the further back you are, the more you can play. SB = default fold.

This isn't just poker strategy. It's decision architecture.

11/ Check your position before every significant move:

① Who acts before me?
② What signals can I observe before I commit?
③ Can I wait, probe, or use private info to improve position?
④ Then: size commitment to match position quality.

Position first. Options second.

# SOP #119 — Twitter Thread Draft

**Posting queue**: ~Feb 2027 (after SOP #118 DualMA Reactivation clears)

---

**Tweet 1/8**
Most people waste weeks after a negotiation falls through.

Not because they don't know what to do next.

Because they don't do it IMMEDIATELY — when the context is still loaded.

SOP #119: Path Closure Option Generation Protocol.

🧵

---

**Tweet 2/8**
The moment a path closes, you have something valuable:

Full working memory of WHY it closed.
Every constraint. Every adjacent option. Every reason it failed.

That context evaporates within hours.

The options you generate NOW will be better than anything you generate next week.

---

**Tweet 3/8**
First: confirm the path is actually closed.

Not "difficult."
Not "probably won't work."
Actually closed — the core condition has been answered NO with no realistic flip.

Premature closure wastes viable paths.
But lingering on a confirmed-closed path wastes the next path.

---

**Tweet 4/8**
G1: Same-session option generation.

Set a 10-minute timer.
Write down 3 alternative paths.
Each must have a concrete first action you could take this week.

No ranking during generation. Just list.

The goal: convert closure energy into exploration energy before it dissipates.

---

**Tweet 5/8**
G2: 2-minute EV triage per option.

3 questions:
- Do I have the skills/resources to try this now?
- What's the best case if it works?
- What's the worst case if it fails?

Priority / Secondary / Defer.

You're not building a plan yet. You're eliminating obvious non-starters.

---

**Tweet 6/8**
G3: First action within 48 hours.

For your top option — one specific action.
Not "research it."
Not "think about it."

"Email X." "Open account at Y." "Schedule call with Z."

Completable in ≤2 hours. Generates new information.

Follow-through rate drops 3x after 48 hours.

---

**Tweet 7/8**
The anti-pattern:

"Let me sleep on it before deciding what to do next."

Processing and option-generation are not mutually exclusive.

Writing the list IS part of processing.

The difference: one leaves you with 3 live options. The other leaves you with 0.

---

**Tweet 8/8**
The pattern I traced back to a 2017 family negotiation that fell through.

Decision confirmed: no equity, no succession path.

Same paragraph: "I'll focus on poker. If that's not enough, tutoring."

No gap. No processing time.

Path closed → options generated → first action.

SOP #01–#119 complete. ✅

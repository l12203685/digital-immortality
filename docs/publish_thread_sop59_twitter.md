**Tweet 2**

Most life systems run on willpower.

Every recurring decision — what to eat, when to exercise, what task to start first — consumes a decision unit.

Not because the problem is hard.

Because no one ever wrote the default.

This is not a discipline failure. It is a system design failure.

**Tweet 4 — G0: Repeated Decision Audit**

Gate 0: Repeated Decision Audit

The rule: any decision that has occurred ≥3 times in 30 days with low outcome variance is a default candidate.

You're not making a new decision. You're replaying the same decision without a script.

Write the script.

Output: default candidate list.

---

**Tweet 5 — G1: Decision Unit Cost Mapping**

Gate 1: Decision Unit Cost Mapping

Not all repeated decisions cost equally.

The multiplier: peak window.

A repeated decision consuming your peak cognitive hours costs 3× more than the same decision in off-peak hours.

Priority rule: any repeated decision in peak window is Priority 1 — regardless of everything else.

---

**Tweet 6 — G2: Default Conversion**

Gate 2: Default Conversion

Each default has four fields:

Trigger: what situation fires this default
Action: exact action — specific enough to execute without deliberating
Override condition: the only situations where live judgment replaces the default
Review date: quarterly — defaults go stale

Validation test: can you execute without making any decision?

If you're still deliberating — the default is underspecified. Rewrite it.

---

**Tweet 7**

Example defaults people don't write but should:

"First sit-down of the day → open single pre-selected priority task, no email for 90 min"

"Non-urgent message arrives → batch response at 12:00 and 18:00 only"

"Meal time → rotation from fixed weekly menu"

None of these require discipline once written.

---

**Tweet 8 — G3: Peak Window Protection**

Gate 3: Peak Window Protection

Track cognitive clarity at 2-hour intervals for 7 days. The 3–4 hour window that consistently scores highest is your peak window.

Rule: peak window = high-cognitive tasks only. No exceptions.

Strategy. Trade decisions. Deep reading. Writing.

Email, admin, logistics — off-peak only. Always.

---

**Tweet 9 — G4/G5: Weekly Scan + Emergency**

Gate 4 (Sunday, 10 min):

How many times did you make a decision that has an active default?
Peak window breached?
New repeated decisions appeared?

Gate 5 (emergency — bypass rate ≥5/week for 2 weeks):

Stop adding new defaults. Find the root cause.
Underspecified trigger? Ambiguous action? Environment friction higher than the non-default path?

If optimal behavior is harder than non-optimal behavior — that is an environment design failure, not a willpower failure.

Fix the environment.

# SOP #12 — Content Distribution & Audience Architecture

> Edward's operating principle: Output = gap detector. Platform = audience density × feedback speed.
> Publishing without distribution = wasted edge. Series #01~#11 exist. Zero reach = zero compounding.

---

## Core Thesis

Content compounds only through distribution. 11 SOPs with zero audience = 11 private memos.
Distribution is not marketing — it is the closing step of the knowledge externalization loop.

---

## The 5-Gate Framework

### G0 — Platform-Audience Fit Test (MD-319)
Before posting anywhere: `audience density × feedback speed ≥ threshold`.
- Twitter/X: trading + systems community ✓ (density HIGH, feedback FAST)
- LinkedIn: career + general systems ✓ (density MED, feedback MED)
- Newsletter: high-intent readers ✓ (density LOW, feedback SLOW — but compounding)
- **Gate**: name the platform, score it on both axes. Skip platforms that score LOW×LOW.

### G1 — Gap Detection Before Content (MD-319)
Every post must answer: "What belief do most people in this audience hold that is wrong or incomplete?"
- Identify the common misconception
- Write toward the gap, not toward comprehensiveness
- **Gate**: one-sentence gap statement must exist before writing the post.

### G2 — Hook = Information Asymmetry (MD-141, Kernel 2)
The hook is the asymmetry signal: "You know something they don't, and they can feel it."
- Bad hook: "Here's what I learned about position sizing"
- Good hook: "95% of traders size positions wrong. The fix takes 5 minutes."
- **Gate**: hook must imply edge. Reader must feel the cost of not reading.

### G3 — One SOP = One Thread = One Claim (MD-116)
Each SOP maps to exactly one thread. Each thread makes exactly one claim.
- Compress the SOP to its single falsifiable claim
- Thread structure: Hook → Problem → Framework (3 gates max) → Self-test → CTA
- **Gate**: thread must be writable in ≤9 tweets. If not, the claim is too broad.

### G4 — Feedback Loop Closure (MD-57)
Post → collect signal → update the SOP or distribution strategy.
- Signal = reply quality, save rate, DM inquiries (not follower count, not likes)
- If no replies after 48h: hook failed → rewrite G2
- If replies are confused: framework too dense → rewrite G3
- **Gate**: every post gets a 48h review. Decision logged in `memory/distribution_log.md`.

### G5 — Compounding Check: Is This Building an Asset? (MD-141)
Distribution compounds when: each post builds audience that amplifies the next post.
Scoring (≥2 required to continue investing in a platform):
- [ ] Follower growth > 0 this week
- [ ] At least 1 inbound DM or collaboration signal this month
- [ ] At least 1 thread saved/bookmarked by someone with domain credibility
- **Gate**: score platforms monthly. Kill or pause platforms scoring 0.

## Failure Modes

| Failure | Diagnosis | Fix |
|---------|-----------|-----|
| Post gets no engagement | Hook failed G2 | Rewrite hook, repost in 72h |
| Follower growth stalls | Wrong platform (G0) | Pivot to higher-density platform |
| Getting likes but no DMs/saves | Content too generic (G1 gap unclear) | Sharpen the misconception |
| Posting exhaustion | No system (G3 violated) | One SOP = one thread. Batch-write 3 threads/week |
| "Nobody cares about trading content" | Sample size too small | Post ≥20 before evaluating platform fit |

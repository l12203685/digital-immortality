# Twitter Thread — SOP #46: Async Communication & Message Triage Protocol

**Domain:** 4 (社交圈) + 8 (生活維護)
**Date:** 2026-04-09 UTC (Cycle 207)
**Hook:** "Every unread message is a decision deferred. You don't have a communication problem. You have a triage problem."

---

**1/**
Every unread message is a decision deferred.

You don't have a communication problem.

You have a triage problem.

🧵

---

**2/**
The actual cost isn't missed messages.

It's decision fatigue:

40 messages/day × 30 seconds each = 20 min/day in micro-decisions.

None of those micro-decisions require your peak cognitive state.

But they erode it.

---

**3/**
The fix: pre-commit everything before you open the inbox.

Who gets a response.
When.
How long.
What you're asking them to do.

This is a triage protocol, not a productivity trick.

---

**4/**
Gate 0: Classify the *source*, not the message.

• T1 (core relationships): respond ≤24h
• T2 (active network): respond ≤48h
• T3 (weak ties): batch once/week
• T0 (cold/mass): archive, no response required

Classify first. Read content second.

Tier determines SLA before the message does.

---

**5/**
Gate 1: Never negotiate SLA in the moment.

If you receive a T1 message and can't respond in 24h:

"Got this, will reply by [time]."

One sentence. No content. Sends the receipt.

Anti-pattern: immediately responding to T0/T3 because it's low-effort.

You're rewarding noise. You're signaling availability.

---

**6/**
Gate 2: Never compose to a blank screen.

For every recurring message type, a template exists.

The second time you write a message type from scratch = system failure.

First instance: acceptable.
Second: create the template.

This is how you get to zero blank-screen composition decisions.

---

**7/**
Gate 3: Message length is inversely correlated with clarity of thinking.

Long draft = you're uncertain about your answer OR uncertain they'll accept it.

If uncertain about the answer → think more, write less.
If uncertain they'll accept → their acceptance isn't in your message. It's in the relationship.

---

**8/**
Gate 4: Every outbound message has exactly one of:

• One clear ask
• One clear offer
• No ask (state it: "no reply needed")

Multi-ask messages split attention and lower response rate on all asks.

One claim per message. If you have 3 points, send 3 messages. Or pick the one that matters.

---

**9/**
Gate 5: Every conversation with ≥1 behavioral signal → extract to durable storage within 24h.

Not a summary. One sentence that could become a principle.

"Samuel said he'd reach out even if there's nothing to discuss" →
`§4: Initiates contact without agenda when relationship value is high`

The conversation didn't count if nothing was extracted.

---

**10/**
Kill condition:

If you're making the same SLA judgment ≥3× for the same sender:

→ You've misclassified them. Reclassify.

The system should make the decision. You should only decide when the system breaks.

---

**11/**
The self-test:

23 unread messages. Among them:
• T1 relationship replied (calibration DM)
• Recruiter cold-message (T0)
• Twitter DM asking for a workbook link (T2)
• Colleague wants your take on their strategy (T2)

Apply the gates.

Total in-the-moment decision time: ~2 min.
Without the system: ~20 min.

Every day.

---

**12/**
You pre-commit trading rules before the market opens.

You pre-commit health defaults so willpower isn't required.

Communication is no different.

Pre-commit your triage criteria. Open the inbox after.

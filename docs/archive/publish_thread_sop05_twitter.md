---

**Tweet 1 (hook)**
Most salary negotiation advice starts too late.

You've already lost by the time they ask for your number.

The error happened 2 weeks earlier — before the meeting, before the research, before you even applied.

Here's the 7-gate SOP that fixes it: 🧵

---

**Tweet 2**
Gate 0: Are you in Exploration or Execution mode?

If you haven't received a real offer with real numbers — you're still in Exploration.

Correct move: open 4–6 parallel tracks simultaneously. Apply, contact, prototype across directions.

"I've done enough research" is NOT a convergence signal. Real signal = concrete offer or demonstrated advantage.

---

**Tweet 3**
Gate 1: Max-loss attribution [最大損失歸屬分析].

Every career path has a worst-case. Before evaluating upside, map the downside:

• Who controls the stop-loss — you or the org?
• Is your personal liability bounded?
• Can you define your own exit condition?

Prefer paths where worst-case loss stays within your own capital and reputation.

---

**Tweet 4**
Gate 2: Write the floor BEFORE the meeting. [書面化底線]

The floor is not a feeling. It's a calculation:

Floor = MAX(
  opportunity cost,
  market percentile (50th–75th),
  personal minimum acceptable
)

Backtrack from take-home [稅後實領]: target ÷ 0.75 = required headline salary.

If the role's ceiling < required headline → path eliminated before you walk in.

---

**Tweet 5**
Gate 2 (continued): Probation terms in writing.

Probation ≠ informal trial. It has salary, conversion conditions, and scope.

If these aren't written: your leverage disappears the moment you sign.

Written floor + written probation terms = the minimum pre-meeting documentation.

---

**Tweet 6**
Gate 3: Diagnose the org type.

Test: Has this org ever responded to individual efficiency gains with pay raises, reduced workload, or promotion?

• YES → Meritocratic. Optimize output; it compounds.
• NO → Compliant [顺从型]. Your efficiency gains go to the org, not you.

In a compliant org: channel surplus capacity into exit preparation, not org benefit.

---

**Tweet 7**
Gate 4: Compound salary viability [薪資成長複合計算].

Don't ask "will I get promoted?" Ask: what annual % increase is required to hit my target by year X?

Formula: (target / current_post_promotion) ^ (1/years) - 1

Then ask: what do the top 10% of peers in this org earn? That's your real ceiling [天花板].

If required % > org's historical max, or your target > peer ceiling → path eliminated. Don't wait to find out in year 3.

---

**Tweet 8**
Gate 5: Build the parallel offer EV table [平行offer機會成本精算].

NEVER evaluate one offer in isolation.

Build the full table before responding:

| Path | Expected value | Max loss | EV |
|------|---------------|----------|----|
| Offer A | | | |
| Offer B | | | |
| Status quo | | | |

One extra rule: if any option involves a domain switch [跨域轉換], find 3 verified cross-domain success cases first — or it stays in Exploration, not the EV table.

"This is my only option" is an information failure, not a constraint.

---

**Tweet 9**
Gate 6: Add knowledge domain externality. [知識外溢價值]

For finance, trading, law, medicine — the salary number is incomplete.

True compensation = salary + (domain_knowledge × personal_capital × applicable_leverage)

Example: Finance role at $800K → domain knowledge on $3M savings at 3% alpha = +$90K/year.

True comp: $890K. Compare domains using this number, not headline salary.

---

**Tweet 10**
The failure modes, ranked:

1. Accepting the first offer before building the EV table (Gate 5 skip)
2. Negotiating without a written floor (Gate 2 skip)
3. Comparing two offers across domains on headline salary alone (Gate 6 skip)
4. Converging on a path because it feels familiar, not because real signal fired (Gate 0 skip)
5. Staying in a compliant org and optimizing output for the org (Gate 3 misdiagnosis)

Each skip is a separate, fixable mistake.

---

**Tweet 11**
The recurring pattern in 300+ career decisions:

The people who win at career transitions don't find better opportunities.

They have better decision architecture:
• Written floors before meetings
• Parallel evaluations, never sequential
• Org type diagnosed before optimizing effort
• Domain knowledge ROI explicitly calculated

Architecture > talent at the margin.

# SOP #40 — Information Asymmetry Edge Protocol

> Kernel 2: 資訊不對稱決定行動方向
> DNA anchors: MD-02/04/16/44/54/57/62

---

## Core Claim

Edge exists only where you have information others don't, interpret information others can't, or act on information others won't. Without asymmetry, any positive EV is noise or temporarily borrowed from the crowd.

**If you can't articulate your informational edge in one sentence, you don't have one.**

---

## Gate Framework

### G0 — Edge Identification (before any position)

State the asymmetry explicitly:
- **Type A: Access edge** — data others don't see (private, hard to obtain, proprietary)
- **Type B: Interpretation edge** — public data, non-consensus framework (quantitative lens, cross-domain synthesis)
- **Type C: Execution edge** — same information, faster or more disciplined action
- **Type D: Behavioral edge** — same information, but others won't act due to structure, emotion, or institutional constraint (MD-04: population exploit)

If none of A/B/C/D applies → **no action**. Bias toward inaction is not cowardice; it's EV preservation (MD-05).

---

### G1 — Asymmetry Decay Check

Information edges decay. Rate of decay depends on:
- Access edge: decays when data becomes public or commoditized
- Interpretation edge: decays when framework is published/widely adopted (MD-61: public indicators → marginal edge → zero)
- Execution edge: decays when algorithms or systems close the latency gap
- Behavioral edge: decays when crowd adapts or regime shifts

**Protocol**: Before acting, ask: _How long is this edge valid?_
- < 1 day: intraday execution only
- 1 week to 1 month: tactical position, tight exit protocol
- > 6 months: structural — size accordingly, defend the moat

---

### G2 — Adversarial Pressure Test

Before committing capital or career capital:
1. **Who is on the other side of this trade/decision?**
2. **Why are they wrong — and what do they know that you don't?**
3. **What if you're the one with the information deficit?**

If you can't answer Q2 → **reduce size by 50%**.
If Q3 scenario is plausible and irreversible → **don't act**.

This is not pessimism — it's the mandatory calibration step before exploiting apparent asymmetry (MD-57: strategy development = define search space first).

---

### G3 — Edge Sizing Rule

Size is a function of edge confidence × edge duration:

```
Position size = base_unit × confidence_score × duration_multiplier

confidence_score: 1.0 (G0 A/B clearly identified), 0.5 (C/D only), 0.0 (none)
duration_multiplier: 2.0 (>6 months), 1.0 (1 week–6 months), 0.3 (< 1 week)
```

Maximum size cap: never exceed 2× base_unit regardless of confidence (tail risk, MD-33).

---

### G4 — Population Exploit Screen

When crowd behavior is detectable and consistent → asymmetry is behavioral:
- Survey: is the consensus position crowded? (short interest, futures positioning, sentiment surveys)
- If >70% of surveyed participants hold the same direction → contrarian signal upgrades to Type D
- Type D edges have highest duration (behavioral inertia changes slowly) — but **require a structural reason** the crowd is wrong, not just contrarianism for its own sake (MD-04)

Checklist:
- [ ] Crowd position is measurable and clearly directional
- [ ] Structural reason the crowd is anchored incorrectly (regulatory constraint, survivorship bias, wrong framework)
- [ ] Exit condition pre-written: if crowd begins unwinding → exit before completion

---

### G5 — Edge Log & Decay Review

Every active edge gets a log entry:

```
Edge ID: E-[date]-[type]
Asymmetry type: A/B/C/D
One-line statement: [who has what information you don't, and why you have the advantage]
Valid until: [date or condition]
Exit trigger: [what event signals decay]
Last reviewed: [date]
```

Quarterly review: if edge statement can no longer be defended → close position, kill SOP, begin G0 again.

## Application Domains

**Trading**: G0 → G3 are primary gates. G4 (population exploit) applies when positioning data available.

**Career**: Edge = domain depth + cross-domain synthesis + network access. G1 decay = when skill becomes commodity (e.g., Excel → Python → LLM).

**Negotiation**: Edge = knowing counterparty's true constraints before they reveal them. G2 adversarial test is mandatory.

**Content**: Edge = publishing frameworks others feel but haven't formalized. G4 decay = when topic becomes mainstream.

---

## Anti-Patterns

- **Fabricated edge**: "I have a gut feeling" — not an asymmetry, not an edge
- **Decayed edge acted on**: edge was valid 6 months ago, still holding the position
- **Undercounting competition**: assuming no one else has reached the same conclusion
- **Contrarianism without structure**: being against the crowd without a reason = noise, not signal

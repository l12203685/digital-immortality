# SOP #34 — Knowledge Monetization Framework

> Convert intellectual capital into revenue before trust decays.
> DNA anchors: MD-111/166/319/320/321/141/108/1

---

## The Problem

You can build a 5,000-follower audience on the strength of 33 free threads.
Zero of them pay you anything.

The default outcome of "build audience first, monetize later" is: later never arrives.
The correct order: proof-of-trust threshold → offer → conversion architecture → reinvestment loop.

---

## Gate 0 — Proof-of-Trust Threshold (before any offer)

**Do not attempt monetization until:**
- ≥3 unsolicited DMs asking for more / how to work with you
- ≥1 thread saved/retweeted by someone with a quant/trading account (peer signal)
- ≥10 replies that demonstrate applied understanding (not "great thread")

**Rationale (MD-321):** Knowledge productization requires the recipient to have a problem.
Engagement quantity ≠ problem density. Wait for pull, not push.

**If threshold not met:** run one "question thread" — "What's your biggest unsolved trading problem?" — before building any product.

---

## Gate 1 — Value Ladder Design

Map offers from zero friction to maximum value:

| Tier | Offer | Price | Friction | Volume |
|------|-------|-------|----------|--------|
| 0 | SOP threads (X) | Free | None | Unlimited |
| 1 | Curated SOP PDF bundle | Free (email gate) | Email | High |
| 2 | 33-SOP annotated workbook | $29–49 | Gumroad | Medium |
| 3 | Live workshop (4hr cohort) | $199–499 | Application | Low |
| 4 | 1:1 strategy review / consulting | $300–500/hr | DM → call | Very low |

**Rule (MD-166 financial freedom three tiers):** Structure so Tier 2 alone covers API costs.
Tier 3 alone covers rent. Tier 4 is discretionary income.

**Anti-pattern:** do not build Tier 4 before Tier 2 is validated.

---

## Gate 2 — Minimum Viable Offer (validate before build)

Before building the workbook or workshop:

1. Post a "soft launch" tweet: *"I'm considering packaging the 33-SOP framework into a workbook. DM me if you'd want this."*
2. Threshold: ≥10 DMs in 48h = validated. Build it.
3. < 10 DMs = insufficient pull. Run G0 question thread first. Do not build.

**Rationale (MD-141 information asymmetry):** You know the value of the framework.
The audience doesn't yet. The DM count measures whether your communication has closed that gap.

---

## Gate 3 — Conversion Architecture

Friction map: reduce the gap between "interested" and "paid":

```
Thread read → "link in bio" → Gumroad page → purchase
             ↓                              
           DM reply → PDF bundle (free) → email list → workshop announce
```

**Required infrastructure:**
- Bio link → single landing page (Gumroad or Notion)
- 1 auto-reply DM template for workbook inquiries
- Email sequence: 3 emails (welcome → SOP sample → workshop invite)

**Anti-pattern:** do not launch paid offer without the free-to-paid path first.
Cold asks convert at <1%. Warmed sequences convert at 5–15%.

---

## Gate 4 — Revenue Stream Anti-Fragility

Minimum viable revenue mix (once all tiers live):

| Stream | Target % of Monthly Revenue |
|--------|------------------------------|
| Workbook sales (Tier 2) | ≥40% |
| Workshop cohorts (Tier 3) | ≥40% |
| Consulting (Tier 4) | ≤20% |

**Rule (MD-108 public indicators edge decays):** Any single-stream revenue model is a single point of failure.
The consulting stream has a bandwidth ceiling (MD-142). Cap it at 20%.

**Anti-pattern:** do not let consulting dominate — it's uncapped time cost, not leveraged income.

---

## Gate 5 — Reinvestment Rate (compounding loop)

```
Month 1–3: 100% reinvest → content tools, paid distribution, community
Month 4–6: 70% reinvest, 30% personal
Month 7+:  50/50 → sustainable cruising speed
```

**Trigger (MD-111 stable income = cognitive resource prerequisite):**
When monthly revenue ≥ API costs × 3: reduce reinvestment pressure.
When monthly revenue ≥ 50% of salary equivalent: Branch 1.4 (alternative income paths) activates.

---

## Self-Test Scenario

**Situation:** 50 threads posted, 800 followers, 5 DMs received, 3 "great thread" replies per post average.

**Apply the SOP:**
- G0: 5 DMs ≥ 3 threshold ✓; 3 applied-understanding replies (low but present) → CONDITIONAL
- G1: Value ladder designed; Tier 2 workbook identified as first paid product
- G2: Run validation tweet before building
- G3: No landing page yet → BLOCKING. Build this first.
- G4: Only one product planned → set roadmap to 3 streams within 6 months
- G5: Reinvestment at 100% until API costs covered

**Decision: BUILD LANDING PAGE → then run G2 validation tweet → then build workbook.**

---

## Kill Conditions

- G2 validation attempt × 2 fails (< 10 DMs each) → pause product build; rerun G0; audit SOP hook quality
- Consulting > 40% of revenue → cap new consulting engagements
- Email list growth < 5/month after 60 days → revise lead magnet

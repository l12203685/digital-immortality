# Twitter Thread — SOP #11: Learning System Design

**Hook:**
Most people optimize *what* they learn.
I optimize the *architecture* of learning.

5 gates. Each one a filter most people skip.
Here's why your study hours aren't compounding:

---

**1/**
Gate 0: Input Quality

Before you open a book or video, ask:
Does this person have skin in the game?
Did they lose money/time/status for getting it wrong?

No? You're learning theory, not knowledge.
Surface-level sources set a low ceiling regardless of hours invested.

---

**2/**
Gate 1: Define the search space first.

Before starting, write:
- What decision will this improve?
- What does "done" look like?
- What's the minimum framework to get there?

If you can't answer → you're browsing, not learning.
Information without a retrieval trigger is just trivia.

---

**3/**
Gate 2: Find the structural analog.

Every new domain has a core mechanism.
Find a domain you already know with the same mechanism.
Map new vocabulary onto the old structure.

I learned options pricing through poker pot odds.
Both are probability × payoff under uncertainty.
Learning speed 3x.

---

**4/**
Gate 3: The Simplicity Test.

After studying, can you state the core in ≤3 rules?
Can you explain it in 60 seconds?
Is your explanation *shorter* than when you started?

If complexity increases → you're memorizing, not understanding.
True understanding compresses.

---

**5/**
Gate 4: Teach the starter kit.

You haven't learned it until you can:
1. Give someone the minimum viable rule to start practicing
2. Name the first edge case they'll hit
3. Point to what they'll want to learn next

Teaching is not completeness.
It's giving the right first move.

---

**6/**
Gate 5: The Compounding Check.

Before deep study, score it:
3 = unlocks 3+ domains you already use
2 = deepens one domain you actively use
1 = new domain, no connection
0 = interesting, no decision impact

Invest at ≥2. Scan at 1. Skip at 0.

Learning compounds like capital.
10% on existing base > 100% on isolated new base.

---

**7/**
Self-test: macro economics for trading.

G0: Ray Dalio (skin in game) ✓
G1: "Which macro regime am I in and what does it favor?" ✓
G2: Analog = poker range construction ✓
G3: 4 quadrants, 4 asset classes, central bank as accelerator ✓
G4: "Start with the 4-quadrant framework, map 2008 and 2022" ✓
G5: Score = 3 (trading + career + capital) ✓

Deep study approved.

---

**8/**
The failure table:

Read 5 books, retained nothing → skipped G1
Can explain but can't apply → skipped G2
Explanation keeps growing → skipped G3
Can't teach anyone → skipped G4
Learned a random domain → skipped G5

One gate missed = whole system leaks.

---

**9/**
Most people's learning system:
Interesting → read → forget → repeat.

This system:
Decision target → quality source → structural analog → compress → teach → compound check.

The difference: one produces trivia. The other produces leverage.

SOP #11 of my Digital Immortality series — how I build systems that survive past me.

**Tweet 2**
The last 5% problem:

The insight exists.
The draft exists.
The thread is formatted.

An unposted draft ≠ content.

---

**Tweet 3**
Why smart creators don't ship:

• "It's not ready" (it's 95% ready)
• "Let me improve the hook first" (infinite)
• "I'll post it tomorrow" (tomorrow never comes)

The fix isn't motivation.
It's a system that makes shipping the default.

---

**Tweet 4**
The 3-layer shipping system:

L1 Execute: daily queue check → format → post → log
L2 Evaluate: weekly posting rate + engagement audit
L3 Evolve: quarterly format rotation based on what works

You can't evolve what you never shipped.

---

**Tweet 5**
G0: Queue Audit (5 min, daily)

3 checks:
1. Is today's post scheduled?
2. Does the thread file exist?
3. Days since last post

Kill switch: ≥14 days since last post → emergency protocol.

Start here. Every day.

---

**Tweet 6**
Non-negotiable shipping budget (G2):

• ≥1 thread per 48h
• Every post logged with date
• Every 10 DMs → check if product launch is triggered
• No new content drafts while ≥3 posts are overdue

Ship before you draft.

---

**Tweet 7**
Emergency protocol (G5):

Trigger: ≥14 days since last post OR 3+ posts overdue.

Step 1: HALT all new drafting.
Step 2: Post oldest overdue thread. Right now. As-is.
Step 3: Post next day. Next day.

Resume drafting only when queue = 0 overdue.

---

**Tweet 8**
The quarterly leverage scan (G3):

Every 90 days:
• Export engagement data
• Rank threads by engagement rate
• Identify which hook format wins
• Run 30-day trial with new format

If ΔEngagement ≥+20% → write as new default.

Optimize what ships. Ship what you optimize.

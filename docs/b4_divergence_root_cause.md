## 1. The Core Split

All 19 divergences trace back to ONE fundamental value hierarchy difference:

| | Edward | Samuel |
|---|--------|--------|
| **Axiom #1** | EV + structural edge → act; no edge → wait | Relationships are the alpha — network density compounds faster |
| **Signal source** | Base rate → structural analysis → gut as tiebreaker | Gut → social proof → numbers as confirming evidence |
| **Default state** | Inaction until edge established | Action bias — move first, optimize later |
| **Social operating rule** | People are assets with expected value and kill conditions | People are the thesis — loyalty overrides exit logic |

**In one sentence:** Samuel runs a relationship-first operating system; Edward runs an EV-first operating system.

## 3. Divergence Pattern Map

### Cluster A: Signal source (5 divergences)
Scenarios: BASE_RATE_CHECK, SOCIAL_PROOF_VS_DATA, EV_OVERRIDE, META_STRATEGY, DOWNSIDE_MODEL

Samuel uses gut → social proof → numbers. Edward uses base rate → structural analysis → action.

**Practical implication:** When Samuel says "3 people I trust recommend X," he treats that as near-final signal. Edward would run a base rate check on X first. Samuel's approach is faster but subject to social proof cascades; Edward's is slower but resistant to groupthink.

**Divergence correction for Edward's twin when simulating Samuel:** Replace "What's the base rate?" with "Who do I trust that has done this?" as the first question.

### Cluster B: Social exit rules (5 divergences)
Scenarios: RELATIONSHIP_DOWNGRADE, FADE_VS_CONFRONT, LOYALTY_EXIT, KILL_CONDITION, SOCIAL_TRUST

Samuel: loyalty overrides exit logic; relationships don't get formally ended.
Edward: kill conditions are pre-defined; exit when triggered regardless of relationship depth.

**Practical implication:** Samuel will maintain relationships that Edward would have already exited. When Samuel says "I still trust X," that statement is more durable than Edward's equivalent. Samuel's social bonds are stickier.

**Divergence correction for Edward's twin when simulating Samuel:** Remove pre-defined kill conditions from social relationships. Replace with "has the frequency dropped naturally?" as the only exit trigger.

### Cluster C: Network theory (3 divergences)
Scenarios: NETWORK_ROI, NETWORK_DORMANCY, SOCIAL_CAPITAL_DEPTH

Samuel: ROI audit — if no reciprocity, actively downgrade; reactivate dormant ties via relational angle.
Edward: DORMANT — tolerates low-activity ties; ROI gate before reactivation.

**Practical implication:** Samuel invests more actively in network maintenance. He will proactively ping dormant contacts; Edward waits for a reason. Samuel's network is warmer but requires more energy investment.

### Cluster D: Social operating speed (3 divergences)
Scenarios: GROUP_DYNAMICS, GROUP_CONTRARIAN, LEARNING_SEQUENCE

Samuel: move first, surface tension early, relationships before skills.
Edward: assess signal/noise first, parallel learning tracks.

**Practical implication:** In group decisions, Samuel anchors early and influences the group's direction. Edward reads the room before committing. Samuel is a faster social mover but more exposed to premature anchoring.

### Cluster E: Gatekeeping and output (3 divergences)
Scenarios: INTRO_GATEKEEPING, INTRO_ASYMMETRY, LEGACY_OUTPUT

Samuel: social relationship overrides quality threshold; visible artifacts signal effort.
Edward: quality standard governs even for close friends; embedded structure over visible artifact.

**Practical implication:** Samuel will make introductions Edward would decline. Samuel will recommend people he trusts even when uncertain about fit. Edward's introductions are higher signal but fewer.

# SOP #84 — Twitter Profile Pre-Launch Audit

> Domain: Distribution / Pre-flight
> Created: 2026-04-09T10:07Z
> Status: OPERATIONAL
> Run: ONCE before posting SOP #01. Then quarterly.

---

## Why This Exists

SOP #83 defines the daily posting ritual. But the ritual assumes the profile is ready.
It's not verified.

**Gap (identified cycle 227 gap scan):**
Bio / link / pinned tweet not confirmed set. Risk: Edward posts SOP #01, profile is blank → zero follow-through → first post wasted.

**This SOP closes that gap. ~15 min total. Run once, then post.**

---

## G0 — Trigger

Run when:
- T1: About to post SOP #01 for the first time
- T2: Quarterly profile refresh (every 3 months)
- T3: Any major pivot in content domain
- T4: Profile stats stagnant for 4+ weeks with no clear content cause

---

## G1 — Profile Core Audit (5 min)

Open Twitter profile. Check each item:

| Element | Requirement | Pass / Fail |
|---------|-------------|-------------|
| Display name | Real name or clear brand identity | ☐ |
| Handle | Consistent with other platforms | ☐ |
| Bio (160 chars) | Contains: who you are + what you post + 1 credibility signal | ☐ |
| Profile photo | Clear, recognizable face | ☐ |
| Header image | Relevant, not default | ☐ |
| Website link | Gumroad page OR best existing link | ☐ |
| Location | Set (increases discoverability) | ☐ |

**Pass threshold: 6/7 or above.**
If <6: fix before posting SOP #01. This is a 5-min fix, not a reason to delay posting.

---

## G2 — Pinned Tweet Audit (5 min)

Pinned tweet = first impression for anyone who visits after seeing a post.

Current pinned tweet: (Edward fills in)

Check:
- [ ] Exists (not blank)
- [ ] Answers "why follow me?" within 2 sentences
- [ ] Has at least 1 concrete credibility signal (years trading, result, system built)
- [ ] CTA: "Follow for weekly SOPs on [topic]" or equivalent

**If pinned tweet fails: create a new one NOW before proceeding.**

Template:
```
I've been building systematic trading + decision frameworks for 9 years.

Every 48h I post 1 SOP — operational protocols distilled from my real process.

If you want frameworks that actually work in practice, follow along.
```

---

## G3 — Bio Sharpness Check (3 min)

Read your bio aloud. Does it answer:
1. **Who**: quantitative trader / systems thinker / builder?
2. **What**: what do you post every 48h?
3. **Why follow**: what's the 1 credibility signal that makes me trust you?

**If any answer is "unclear" — rewrite bio before posting.**

Bad bio: "Trading | Systems | Learning every day"
Good bio: "Quantitative trader. 9 years building systematic strategies. Posting 1 operational SOP every 48h — how my trading brain actually works."

---

## G4 — Link Destination Check (2 min)

Click the link in your profile. Verify:
- [ ] Link loads (not 404)
- [ ] Destination matches what bio promises
- [ ] If no Gumroad yet: link to best alternative (newsletter, GitHub, LinkedIn)

**If no link: add any credible landing page. Blank link = 0 conversion from profile visits.**

---

## G5 — Pre-Post Checklist Sign-Off

Before running SOP #83 G1 (pick today's post):

- [ ] G1 profile audit: 6/7 pass
- [ ] G2 pinned tweet: exists + CTA
- [ ] G3 bio: answers who/what/why
- [ ] G4 link: loads + relevant
- [ ] **Sign-off: profile is launch-ready**

Once signed off → run SOP #83 → post SOP #01.

---

## Anti-patterns

| Pattern | Why it fails |
|---------|-------------|
| "I'll fix the profile after posting" | First post drives profile visits; blank profile = wasted traffic |
| "Bio sounds good to me" | Read it as a stranger. If bio doesn't convert in 5 seconds, rewrite. |
| "I'll add the link when Gumroad is ready" | No link = no CTA = no conversion path. Add any link now. |
| "Pinned tweet is optional" | It's the first thing visitors see. Not optional. |

---

## Quarterly Refresh Cadence

After launch:
- Run G1–G5 again at: 2026-07-09, 2026-10-09, 2027-01-09
- Check what the most-engaged post is → update bio to reflect it
- Update pinned tweet if a post hit >500 impressions with strong engagement

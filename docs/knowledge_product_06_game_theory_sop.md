# SOP #06: 賽局決策框架 (Game Theory Decision Framework)
> UTC: 2026-04-08T20:00Z | Cycle 122 | Branch 7 (知識輸出)
> Backing MDs: MD-295, MD-297, MD-304, MD-305, MD-306, MD-325, MD-326

## Overview

任何競爭場景（撲克/談判/職場/投資）都有共同結構：
- 參與者有角色和勝利條件
- 信息不完全
- 行動影響後續行動
- 對手有可剝削的偏差

本 SOP 是通用賽局決策框架，6 個 Gate 依序執行。

---

## G0: 識別賽局結構 (Identify Game Structure)

**問：這是什麼類型的賽局？**

| 維度 | 選項 | 影響 |
|------|------|------|
| 參與者數量 | 2人 vs 多人 | 多人=equity門檻倍增 (MD-305) |
| 零和/正和 | 零和（撲克/談判）vs 正和（合作） | 正和=引入外部資金 (MD-93) |
| 信息類型 | 完全/不完全 | 不完全=建模對手range |
| 時間結構 | 單次 vs 重複 | 重複=聲譽/報復成本列入EV |

**Gate 通過條件**：能回答「誰在場、目標是什麼、信息有多不完全」。

---

## G1: 確認角色+勝利條件 (Confirm Role & Win Conditions)

**問：我的角色是什麼？我的勝利條件是什麼？**

- 明確自己的角色（攻/守/中立）
- 寫下具體勝利條件（而非模糊的「贏」）
- 確認勝利條件在當前局面下是否仍然有效

**陷阱**：在談判中把「讓對方說Yes」當勝利條件，而非「達到自己的底線+」。

**Gate 通過條件**：勝利條件能用一句話精確表達且可量化。

---

## G2: 最大威脅識別 (Identify Biggest Threat First) ← MD-325

**問：我最怕對手出現什麼特徵？**

操作步驟：
1. 從自己角色視角，列出「如果對方是X，我勝率最低」的特徵清單
2. 掃描場上參與者，確認是否有符合威脅畫像的對手
3. 存在高危對手時：優先中和（隔離/規避/提前攻擊）

**範例（阿瓦隆）**：我是好人方 → 最怕壞人方有「邏輯清晰+能讀人+主動帶節奏」的玩家 → 進場後優先識別誰符合此畫像。

**Gate 通過條件**：威脅清單已建立，場上是否存在高危對手已確認。

---

## G3: GTO 基線建立 (Establish GTO Baseline) ← MD-295

**問：在完全無信息的情況下，最優混同策略是什麼？**

- GTO（Game Theory Optimal）= 對手無法通過改變策略從你身上獲益的均衡
- 防禦率公式：如果對手下注 B 到底池 P，你的最低跟注率 = B / (B + P)（避免對手純bluff獲益）
- GTO基線作為「預設值」，後續根據對手偏差調整

**陷阱**：在多人底池中使用頭對頭 GTO = equity 門檻低估 (MD-305)。

**Gate 通過條件**：能計算當前場景的GTO基線行動（即使後續要偏離）。

---

## G4: 剝削偏差識別 (Identify Exploitable Deviations)

**問：對手在哪裡偏離了GTO？我應該往哪個方向偏離去剝削？**

- 觀察對手的行為模式（frequency分析）
- 識別「過高」或「過低」的頻率（例：c-bet頻率在濕板偏高 = 過度bluff）
- 剝削方向：對方過度bluff → 提高call率；對方過度value → 提高fold率

**邊界條件**：剝削偏差會讓自己也失去GTO保護，需確認對手能持續保持偏差（短局=剝削；長局=回歸GTO）。

**Gate 通過條件**：有具體觀察數據支持偏差識別，而非主觀印象。

---

## G5: 執行+部位管理 (Execute & Position Management) ← MD-326

**問：執行前，摩擦成本算清楚了嗎？**

- **執行前**：確認來回摩擦成本（手續費/機會成本/聲譽成本）< 預期EV
- **部位大小**：從最大可承受虧損反推（不從期望報酬正推）
- **持倉管理**：趨勢中加碼不減碼 (MD-49)；勝率信號持續監控

**交易成本SOP（MD-326）**：
```
來回成本 = 手續費率 × 成交金額 × 2
進場條件 = 預期EV > 來回成本（確定值）
```

**Gate 通過條件**：摩擦成本已算清，部位大小已從虧損上限反推。

## Self-Test Scenario

**場景**：你正在和兩家公司談 Offer。A 公司已出 offer（年薪80萬），B 公司進入最終面試。你最想去B，但A要求你本週給答覆。

用本框架走一遍：

- **G0**: 薪資談判 + 職涯選擇；兩家競爭；不完全信息（B還未出offer）；重複賽局（職涯聲譽）
- **G1**: 我的角色=求職者；勝利條件=拿到B的offer且薪資≥90萬，或確保A的條件在週五前提升
- **G2**: 最大威脅=A在我拒絕後不撤offer / B因拖延取消面試 → 先確認A是否有彈性，再跟B要加速
- **G3**: GTO基線=誠實告知A有其他流程，要求1週延期；不透露B的存在
- **G4**: A的偏差=急於成交（本週要求答覆=他們有urgency）→ 剝削方向=要求延期或要求對方先提升條件
- **G5**: 摩擦成本=拒絕A的機會成本（80萬的確定offer）；EV(等B) = P(B出offer) × 90萬 + P(沒出) × 0 → 若P(B)>0.89，等B是正EV

**正確行動**：要求A延期至少一週（G3 GTO基線），同時請B加速流程（G2 威脅中和）。

## 適用場景清單

| 場景 | G0關鍵識別 | G2最大威脅 |
|------|-----------|-----------|
| 撲克 | 零和多人，不完全信息 | 有GTO基礎+快速讀人的玩家 |
| 薪資談判 | 不完全信息，重複賽局（聲譽） | 對方完全不透露標準=信息劣勢 |
| 合作談判 | 正和，對稱信息偏低 | 對方隱藏BATNA |
| 競品競爭 | 重複賽局，多方 | 有資本優勢且願意打價格戰的對手 |
| 職場政治 | 重複賽局，多方，高信息不確定 | 能影響決策者且動機不透明的人 |

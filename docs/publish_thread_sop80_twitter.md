Tweet 2:
The failure mode isn't boot test failure.

It's stable boot tests while the underlying reasoning drifts.

The system passes because it learned to output the expected string — not because it internalized the principle.

This is harder to detect than a failed test.

---

Tweet 3:
3 calibration checks you can't automate:

1. Derivation chain: does the system walk through the logic, or just output the answer?
2. Novel scenarios: does it apply principles to situations it hasn't seen?
3. Edge cases: does it recognize when a principle doesn't apply?

Deterministic tests can't catch any of these.

---

Tweet 4:
The calibration protocol:

Monthly. 3 LLM-required scenarios (formula application + option enumeration).
Expected: MISALIGNED on first pass without derivation cues.
Fix: re-read DNA section, not just the answer.

If LLM passes all 3 without cues: either too easy or the scenarios are stale. Replace them.

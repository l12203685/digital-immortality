# Twitter Thread — SOP #35: Knowledge Compounding & Skill Half-Life Protocol

---

**Tweet 1/**
> You consumed 50 books last year. Can you teach any of them?
>
> If not, you didn't learn — you optimized for the feeling of learning.
>
> Here's the 6-gate system that separates knowledge that compounds from knowledge that evaporates. 🧵

---

**Tweet 2/**
> Most learning fails at the architecture level, not the content level.
>
> No skeleton → fragments you can't retrieve.
> No output → consumption, not integration.
> No transfer test → facts, not frameworks.
>
> You can read for a decade and compound toward zero.

---

**Tweet 3/**
> The default learning loop:
>
> Read → feel smart → next book → forget in 3 weeks.
>
> The problem: no retrieval structure, no output gate, no transfer test.
>
> Knowledge without a slot to fit into is noise with a highlight marker on it.

---

**Tweet 4/**
> Gate 0: Build the skeleton BEFORE absorbing any detail.
>
> Before opening the first source:
> — Write the TOC of what you expect to learn
> — Name the 3–5 load-bearing concepts
> — Map one analogy to a domain you already own
>
> 3h in and can't draw a concept map? Stop. Build it first.

---

**Tweet 5/**
> Gate 1: Input quality sets your ceiling. Effort cannot compensate.
>
> Audit every source:
> — Practitioner or theorist?
> — Primary source or summary-of-summary?
> — Skin in the game?
>
> A trader's 5-page memo outperforms an academic's 400-page textbook.
> Fix the input first.

---

**Tweet 6/**
> Gate 2: After every learning session, produce one output.
>
> Choose one:
> — A decision rule: "In X, do Y because Z"
> — A tweet draft explaining the concept
> — A test prediction in a real domain
>
> Kill condition: no output for >7 days → invoke this gate before any further input.

---

**Tweet 7/**
> Gate 3: Can you apply this to a completely different domain?
>
> If yes → framework. Compounds.
> If no → fact. Useful but local.
>
> Test: apply option theory to salary negotiation.
> Optionality, time value, exercise decision, volatility.
>
> No non-trivial insight → downgrade to fact.

---

**Tweet 8/**
> Gate 4: Not all skills decay at the same rate.
>
> Tools & syntax: ~3yr half-life
> Domain techniques: ~5–7yr
> Strategic frameworks: ~10–15yr
> Mental models: permanent
>
> Audit quarterly. Archive or schedule.
> Don't assume a skill you haven't used in 90 days is still current.

---

**Tweet 9/**
> Gate 5: If you can't write a teachable SOP for it, you don't own it.
>
> Productizing forces every implicit step to surface.
> Wherever you stall writing the SOP → that is the gap.
>
> Can't write even 3 gates after substantial study?
> Return to G0. The learning wasn't structural.

---

**Tweet 10/**
> Scenario: 3 weeks on option theory. Can explain Black-Scholes. Can't say where you'd enter a trade.
>
> G0: No skeleton → draw the concept map now.
> G2: No output → write one decision rule before reading another page.
> G3: Apply to salary negotiation → nothing → downgrade to fact.

---

**Tweet 11/**
> Knowledge = time × density.
>
> Density = input quality × skeleton × output forcing × transfer depth.
>
> Low-density inputs compound toward zero.
> High-density inputs with structure compound permanently.
>
> Most people optimize volume. The variable that matters is density.

---

**Tweet 12/**
> The 6-gate system:
>
> G0: Skeleton before detail
> G1: Input quality sets ceiling
> G2: Output after every session
> G3: Cross-domain transfer = internalized
> G4: Audit skill half-life quarterly
> G5: Can't SOP it = don't own it
>
> Save this. Run it on your last subject.
> Follow for SOP #1–35.

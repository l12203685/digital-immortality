**Tweet 2/8**

• `branch-2.2` (what branch is that?)

**Tweet 3/8**
The 5-category taxonomy that survives long-term:

1. behavioral-patterns — how you decide
2. self-awareness — what you miss, how you correct
3. methodology — your processes
4. domain-knowledge — what you know about specific fields
5. hypotheses — untested beliefs

Simple. Non-overlapping. Survives cold starts.

---

**Tweet 4/8**
What decays the taxonomy:

→ Branch labels (branch-1.1 = provenance, not meaning)
→ Tool names (SOP-92 = pointer, not concept)
→ "domain-knowledge" as a catchall for everything
→ Adding new tags without asking: "which of the 5 does this fit?"

Each shortcut makes the next cold start more expensive.

---

**Tweet 5/8**
When to evolve the taxonomy (G0 triggers):

T1: >50 insights since last audit
T2: New insight doesn't fit any category
T3: Any category exceeds 10 insights (split candidate)
T4: Two categories share >50% content (merge candidate)
T5: System flags taxonomy as neglected

---

**Tweet 6/8**
Split/merge rules:

>10 insights → define 2 subcategories with non-overlapping definitions
>50% content overlap → merge + redefine
3+ no-fit insights → create new category (with crisp definition)

Key: split/merge is a design decision, not an editorial one.
The definition comes first. Reclassification follows.

---

**Tweet 7/8**
Drift detection at every 50-insight milestone:

HEALTHY: 0 unclassified, no category >15, narrative gap <30
WATCH: 1-5 unclassified, any category 10-15
DRIFT: >5 unclassified, any category >15, narrative gap >60
CRITICAL: no canonical categories applied at all

Current system audit: DRIFT (narrative gap=113 insights)

**Tweet 8/8**
The meta-lesson:

A taxonomy audit is a system audit.

If your tags have drifted, your retrieval has drifted.
If your retrieval has drifted, your cold starts are degraded.
If your cold starts are degraded, the system loses fidelity each restart.

SOP #116 closes the drift loop before it becomes behavioral decay.

# SOP #11 — Learning System Design & Knowledge Compounding

> **Meta-SOP**: This governs how all other SOPs are built.
> Better learning → better everything else.
> Source MDs: 55, 57, 116, 120, 139, 141, 300, 322 + supporting

---

## Core Claim

Most people optimize the *content* of learning.
Edward optimizes the *architecture* of learning.

The ceiling is set by **input quality**, not study hours.
Internalization is proved by **cross-domain transfer**, not recall.

---

## The 5 Gates

### G0: Input Quality Gate
**Rule:** Before allocating time, audit the source.
- Who is teaching? Do they have skin-in-the-game? (MD-135: 實操知識=skin-in-the-game才外流)
- What incentives shape what they're saying?
- Is this primary source or summary-of-summary?

**Test:** Can you trace the claim back to someone who lost money/time/status getting it wrong?
If no → deprioritize. Surface-level sources set a low ceiling regardless of time invested. (MD-139)

**Anti-pattern:** Reading 10 summaries instead of 1 source. Each layer removes precision and adds noise.

---

### G1: Search Space Before Content
**Rule:** Define what you're trying to learn *before* you learn it. (MD-57: 策略開發先定搜尋空間)

Write down:
1. What decision will this knowledge improve?
2. What would "done" look like? (Specific output, not "understand X")
3. What's the minimum viable framework to get there?

**Test:** Can you write the target output format before starting?
If no → you're browsing, not learning.

**Why this matters:** Without a defined search space, you accumulate knowledge with no retrieval trigger.
Information only becomes useful when it has a slot to fit into.

---

### G2: Find the Structural Analog
**Rule:** Every new domain has a structural analog to something you already know. Find it first. (MD-55: 框架跨域=真正內化)

Process:
1. What is the *core mechanism* of this domain? (e.g., poker = incomplete information + range vs range)
2. Which domain you already know has the same core mechanism?
3. Map the new vocabulary onto the old structure.

**Test:** Can you explain this domain's key insight using metaphors from a domain you mastered?
If yes → internalized. If no → still surface-level.

**Example:** Options pricing → poker pot odds (both are probability × payoff under uncertainty).
Once mapped, every options concept has a poker analog. Learning speed 3×.

---

### G3: Simplicity Test
**Rule:** True understanding produces simplicity, not complexity. (MD-116: 簡單才有效=深度理解的輸出)

After studying, ask:
- Can you state the core in ≤3 rules?
- Can you explain it to someone in 60 seconds?
- Is your explanation shorter than when you started?

**If complexity increases → you're memorizing, not understanding.**

The goal is compression: take complex inputs, output a simple decision rule.
Complex explanations signal incomplete internalization.

---

### G4: Output-Test (Teaching Gate)
**Rule:** You haven't learned it until you can teach the starter kit AND anticipate 3 extensions. (MD-120: 教學=給起手式讓人自己思考延伸)

Teach format:
1. Here's the minimum viable rule to start practicing
2. Here's the first edge case you'll hit
3. Here's what you'll want to explore after that

**If you can't do this → review G2 and G3 first.**

Teaching is not about completeness. It's about giving someone the *right first move*.
The right first move is a lagging indicator of true understanding.

---

### G5: Compounding Check
**Rule:** Before allocating deep study time, ask: does this multiply existing knowledge?

Score:
- **3**: Unlocks applications across 3+ domains I already use
- **2**: Deepens one domain I actively use
- **1**: Adds a new domain that doesn't connect to existing work
- **0**: Interesting but no clear decision impact

**Threshold:** Score ≥ 2 → invest. Score 1 → light scan only. Score 0 → skip.

**Why:** Learning compounds like capital. 10% on existing base → more value than 100% on an isolated new base.
Meta-knowledge (decision frameworks, probability, game theory) compounds across all domains. (MD-141: 思維層級=比對手多一層)

## Self-Test Scenario

**Situation:** You want to learn about macro economics to improve your trading.

Run the gates:
- **G0**: Source = Ray Dalio's *How the Economic Machine Works* (primary, skin-in-the-game) ✓
- **G1**: Target = "Can I identify which macro regime I'm in and what asset class it favors?" ✓
- **G2**: Analog = Poker range construction — macro regime = opponent range; positioning = bet sizing ✓
- **G3**: Core in 3 rules: (1) growth up/down × inflation up/down = 4 regimes, (2) each regime favors different assets, (3) central bank policy = regime accelerator ✓
- **G4**: Starter kit = "Learn the 4-quadrant framework first, then map 2008 and 2022 onto it" ✓
- **G5**: Score = 3 (applies to: trading regime selection, career timing, capital allocation) ✓

**Result:** Deep study approved. Full G5 pass.

## Common Failure Modes

| Failure | Diagnosis | Fix |
|---------|-----------|-----|
| Reading 5 books, retaining nothing | Skipped G1 (no target output) | Define decision before starting |
| Can explain but can't apply | Skipped G2 (no structural analog) | Find the isomorphic domain |
| Explanation keeps getting longer | Skipped G3 (memorizing not understanding) | Force 3-rule compression |
| Learned X but can't teach anyone | Skipped G4 (no output test) | Find someone to explain it to |
| Know a lot about a random domain | Skipped G5 (no compounding check) | Score domains before investing |

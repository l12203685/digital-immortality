# Twitter Thread: SOP #53 — Cognitive Performance & Decision Bandwidth
> UTC: 2026-04-09 | Queue position: #53 | ~Jul 22

---

1/ Your worst decisions aren't made from lack of knowledge.

They're made tired, distracted, and in the wrong time slot.

Cognitive bandwidth is the scarce resource no one manages.

Here's my system. 🧵

---

2/ The core insight:

The same decision made at 9am vs 4pm is a different decision.

Not because facts changed. Because *you* changed.

Peak cognitive window = first 4h after waking.
Budget: 20 units/day. Peak: 12.

Guard it like capital.

---

3/ G0: Weekly bandwidth audit.

4 tiers:
- L0 Autopilot: habits, physical maintenance (0 units)
- L1 Routine: email, templates, reviews (1-2 units)
- L2 Focused: code, writing, systems (4-6 units)
- L3 Deep: novel problems, calibration, high-stakes decisions (8-10 units)

Rule: ≥1 L3 task must land in peak window. Non-negotiable.

---

4/ G1: Classify before scheduling.

Before adding ANY task to calendar, ask:
L0/L1/L2/L3?

If L2+L3 > 12 units in peak window → drop L2.
Never drop L3.

Kill condition: ≥3 L1 tasks in peak window = system failure. Clear them immediately.

---

5/ G2: Decision debt kills bandwidth silently.

Every open "should I...?" or "when do I...?" runs in the background.

More than 5 open decisions = cognitive debt emergency.

Weekly audit: decide immediately if possible, or set an explicit trigger date. Delete undecidable ones.

---

6/ G3: The recurring-decision kill.

Same low-value decision 3+ times per week?

Don't make better decisions. Eliminate the decision.

Pre-commit a default. System-design it away.

This is how morning_defaults.md was born. 171 min/week recovered.

---

7/ G4: The irreversibility gate.

Before any strategy kill, career move, capital reallocation, relationship downgrade:

1. >4h since wake? → defer to tomorrow's peak
2. >3 deep tasks done today? → defer
3. Sleep <6h last night? → recover first (SOP #52), then decide

All three are AND gates. One trigger = no decision today.

---

8/ G5: Emergency protocol.

When decision debt >5 OR peak window is lost:

1. Stop all deep work
2. List every open decision (5 min)
3. Decide or park each with a trigger date (10 min)
4. Block tomorrow morning as recovery
5. Run G3 on any recurring pattern

---

9/ Domain map:

- Trading decisions → L3, peak only. Signal monitoring → L1, automated.
- DNA calibration → L3, scheduled explicitly.
- Salary negotiation prep → L3, never rushed.
- Communication triage → L1, off-peak only (SOP #46).
- Physical health → L0, habits not decisions (SOP #52).

---

10/ Self-test: 15:00, 2h free. DualMA kill decision pending. 23 unread messages.

G4: strategy kill = L3 + post-peak + no threshold crossed → defer to 09:00 tomorrow.
SOP #46: 23 messages = L1, post-peak = correct slot → triage now.

One decision. Zero cognitive waste.

---

11/ The meta-insight:

Cognitive performance isn't about working harder in peak hours.

It's about not wasting peak hours on L1 work.

Most people do the opposite: urgent L1 floods the morning. Deep work gets the scraps.
